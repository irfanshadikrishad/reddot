# RedDot Privacy Policy Summary

RedDot stores safety data on the device first. Safety plans, journal entries, trusted contacts, reminders, and app-lock settings are kept locally and protected with encryption where appropriate. The app does not upload these records to a RedDot backend because no such backend exists.

Firebase Authentication is used only for email/password sign-in, email verification, password reset, and sign-out. That identity traffic is handled by Firebase, but the app does not place safety records in Firebase.

The app may ask for permission to use local device services such as the dialer, SMS composer, location, notifications, camera, or file picker when the user starts an action that needs them. Location is used only for the active action and is not retained after the flow ends.

Exported files, shared text, and anything opened in the system share sheet leave RedDot protection and are governed by the destination app or file system.
